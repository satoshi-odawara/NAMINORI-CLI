# データ分析依頼ヒアリングシート (v1.1)

## 1. 分析の目的・背景 (Goal)

- **解決したい課題:** Jリーグの観客動員数予測モデルを構築し、動員数に影響を与える主要な要因を特定したい。
- **アウトプットの活用イメージ:** 試合ごとの観客数予測に基づいたスタジアム運営（スタッフ配置、グッズ発注等）の最適化、および効果的な集客施策の立案。

## 2. データに関する情報 (Data Metadata)

- **対象ファイル名:** train.csv, condition.csv, stadium.csv
- **データの内容・構造:** - 内容: 試合情報（日時、カード、スタジアム、天候、スコア、出場選手等）と観客数(y)
  - 書式: CSV/ヘッダーあり/UTF-8
  - カラム定義: train.csv(id, y, year, stage, match, gameday, time, home, away, stadium, tv) 等
- **データの緒言:** - サンプリング周波数: 1試合1レコード
  - 測定期間・量: 2012年〜2014年（train.csv）

## 3. ドメイン知識・制約条件 (Domain Knowledge)

- **既知の情報（正常・異常の目安）:** 観客数はスタジアムのキャパシティ(capa)を超えない。週末やナイトゲーム、人気チーム同士の対戦で増加する傾向がある。
- **特注事項（注目ポイント）:** 天候の影響、特定のチーム（例：浦和レッズ等）の集客力、地理的な近さ（ダービーマッチ）。
- **対象装置・環境の情報:** Jリーグ（J1/J2）

## 4. 成果物の要望 (Output Requirements)

- **希望する分析手法:** 回帰分析、特徴量重要度分析。
- **保存が必要な資産:** レポート(MD)、解析スクリプト(PY)、前処理済みデータ(CSV)
