案件固有ルール (Project Specific Rules) - Jリーグ観客動員数予測

本ドキュメントは、gemini.md 技術憲章に基づき、Jリーグ観客動員数予測案件に固有の論理・物理的制約を定義する。標準規約（rules/）から隔離されたドメイン知識のみをここに集約する。

1. 物理的妥当性・制約条件

  • 観客数の上限: 観客数 (y) は、スタジアムのキャパシティ (capa) を決して超えない。
  • 観客数の下限: 観客数 (y) は 0 以上の整数である。
  • 開催日: gameday は J1/J2 のリーグ戦スケジュールに基づく。

2. ドメイン知識（仮説・注目ポイント）

  • 集客力の高いチーム: 特定の人気チーム（例：浦和レッズ等）の試合は動員数が増加する。
  • ダービーマッチ: 同一地域、または歴史的ライバル関係にあるチーム同士の対戦は動員数が増加する。
  • 日程の影響: 週末（土・日・祝）およびナイトゲームは、平日のデイゲームよりも動員数が増加する傾向にある。
  • 天候の影響: 雨天や荒天は動員数を減少させる負の要因となる。

3. 評価指標の基準

  • ターゲット指標: RMSE (Root Mean Squared Error)
  • 目標精度: 先行研究やベースライン（前回の記録 RMSE: 3549）を参考に、物理的に妥当な予測誤差を目指す。

4. カテゴリ化定義

  • 天候の簡略化: weather カラムを、雨、曇、晴、雪などの主要カテゴリに集約する。
  • ステージ定義: stage は J1 または J2 を指す。
