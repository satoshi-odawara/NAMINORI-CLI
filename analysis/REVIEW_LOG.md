# REVIEW LOG - Jリーグ観客動員数予測

## 1. 分析の教訓（ドメイン知識）
- **物理的上界の重要性**: スタジアムキャパシティが集客の天井を決めるため、モデルの入力に含めることは必須である。
- **リーグカテゴリの支配力**: J1 と J2 では動員ベースが全く異なるため、単一モデルで扱う場合はフラグ化が極めて有効。

## 2. プロセス・規約の改善（標準規約への昇華）
- **隔離の有効性**: `constants.py` への隔離により、EDA や前処理スクリプトから「特定のチーム名」を排除でき、可読性と再利用性が向上した。
- **レポートの三層構造**: 経営向けサマリーを冒頭に置くことで、技術的な詳細（RMSE等）に埋もれない価値提供が可能になった。

## 4. 規約遵守に関する教訓
- **資産形式の完全性**: 標準規約 (`report_rules.md`) に規定された HTML 出力を失念し、MD のみで完了としたことは重大なプロセス不備であった。
- **対策**: 今後は各フェーズの完了時に `utils/report_generator.py` を実行し、HTML 成果物が生成されていることを [QA] が目視確認することを義務付ける。
