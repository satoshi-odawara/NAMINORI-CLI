# PHASE 1: データ理解・EDA レポート

## 1. エグゼクティブサマリー
- **背景**: Jリーグ観客動員数予測の基盤構築のため、初期データの品質と物理的整合性を確認した。
- **目的**: データの欠損状況、外れ値、および物理的制約（キャパシティ）の遵守を検証する。
- **結果**: 欠損値はなく、全てのデータがスタジアムキャパシティの範囲内に収まっていることを確認。1試合のみ観客数0（id: 15699）が存在する。
- **アクション**: 次フェーズにて、`gameday` から曜日・祝日フラグを抽出し、動員数への影響を数値化する。

## 2. 現場担当者向け分析（ビジュアル＆アクション）

### 2-1. 観客動員数 (y) の分布
- **発見**: 観客動員数は 0 から 54,905 人の範囲で分布しており、平均は約 10,662 人。
- **So What?**: J1 と J2 では動員規模が大きく異なるため、モデル構築時には `stage` を重要な説明変数として扱うべきである。

![J1 vs J2](figures/y_by_stage.png)

### 2-2. 物理境界チェック
- **発見**: `y > capa` となるデータは 0 件。
- **解読ガイド**: 物理的にあり得ない「キャパ超え」のデータが存在しないことを確認した。

![Capa vs y](figures/y_vs_capa.png)

## 3. エンジニア向け算出根拠
- **結合ロジック**: `train.csv` + `condition.csv` (on id) + `stadium.csv` (on stadium/name)。
- **無観客試合の処理**: id: 15699 については、学習時の外れ値として扱うか、フラグ立てによる制御を検討する。
